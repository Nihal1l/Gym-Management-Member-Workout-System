## üìã Deployment Steps

### Step 1: GitHub Setup
- [ ] Create GitHub account (if needed)
- [ ] Create new repository: `gym-management-api`
- [ ] Add GitHub remote: `git remote add origin https://github.com/USERNAME/gym-management-api.git`
- [ ] Push code: `git push -u origin main`

### Step 2: Render Setup
- [ ] Create Render account at https://render.com
- [ ] Create new Web Service
- [ ] Connect GitHub repository
- [ ] Configure build and start commands
- [ ] Create PostgreSQL database service
- [ ] Connect database to web service
- [ ] Set environment variables

### Step 3: Environment Variables on Render
- [ ] `DEBUG=False`
- [ ] `SECRET_KEY=[auto-generated by Render]`
- [ ] `ALLOWED_HOSTS=your-service-name.onrender.com`
- [ ] `DB_ENGINE=django.db.backends.postgresql`
- [ ] Database connection variables (auto-set by PostgreSQL service)

### Step 4: Deployment
- [ ] Trigger initial deployment
- [ ] Wait for build to complete
- [ ] Check deployment logs for errors
- [ ] Verify "Status: Live" (green)
- [ ] Note your public URL

### Step 5: Post-Deployment Testing
- [ ] Test login endpoint
- [ ] Test GET requests
- [ ] Test POST requests (create resources)
- [ ] Verify admin panel works
- [ ] Check API documentation page
- [ ] Test with Postman collection

## ‚ö†Ô∏è Common Issues & Fixes

### Build Fails
- Check build logs
- Verify `requirements.txt` syntax
- Ensure all dependencies are listed

### 502 Bad Gateway
- Check web service logs
- Verify database connection
- Ensure migrations ran successfully

### Static Files Not Found
- Not critical for API-only backend
- If needed, run `python manage.py collectstatic`

### Database Connection Error
- Verify PostgreSQL service is running
- Check connection string in environment variables
- Ensure `DB_ENGINE=django.db.backends.postgresql`
